# عرض تقديمي لمشروع نظام إدارة المستشفيات

## نظرة عامة على المشروع

- **اسم المشروع**: نظام إدارة المستشفيات
- **الهدف**: تصميم وتنفيذ نظام قاعدة بيانات متكامل لإدارة المستشفيات باستخدام Oracle SQL*Plus
- **النطاق**: يغطي النظام إدارة المرضى، الأطباء، الممرضين، الأقسام، الغرف، المواعيد، الإقامة، الأدوية، الوصفات الطبية، الفحوصات المخبرية، والفواتير

## مخطط العلاقات بين الكيانات (ER Diagram)

### الكيانات الرئيسية:
- المرضى (Patients)
- الأطباء (Doctors)
- الممرضون (Nurses)
- الأقسام (Departments)
- الغرف (Rooms)
- المواعيد (Appointments)
- الإقامة في المستشفى (Admissions)
- الأدوية (Medications)
- الوصفات الطبية (Prescriptions)
- تفاصيل الوصفة (PrescriptionDetails)
- الفحوصات المخبرية (LabTests)
- نتائج الفحوصات (TestResults)
- الفواتير (Bills)
- تفاصيل الفاتورة (BillDetails)

### العلاقات الرئيسية:
- علاقات واحد إلى متعدد (1:N): مثل علاقة المرضى بالمواعيد
- علاقات متعددة إلى واحد (N:1): مثل علاقة الأطباء بالأقسام
- علاقات واحد إلى واحد (1:1): مثل علاقة الأقسام برؤساء الأقسام
- علاقات متعددة إلى متعددة (N:M): مثل علاقة الوصفات بالأدوية

## النموذج العلائقي (Relational Schema)

### تحويل مخطط ER إلى نموذج علائقي:
- تم تحويل كل كيان إلى جدول
- تم تحديد المفاتيح الأساسية والأجنبية
- تم حل العلاقات متعددة إلى متعددة (N:M) من خلال جداول وسيطة
- تم تطبيق قواعد التطبيع حتى الشكل الطبيعي الثالث (3NF)

## تنفيذ قاعدة البيانات

### إنشاء الجداول (DDL):
- تم إنشاء 14 جدولاً مترابطاً
- تم تحديد المفاتيح الأساسية والأجنبية
- تم إضافة قيود للتحقق من صحة البيانات
- تم إنشاء مشغلات (Triggers) لتنفيذ قواعد العمل

### إدخال البيانات (DML):
- تم إدخال بيانات عينة كافية (5 سجلات على الأقل لكل جدول)
- تم التأكد من سلامة البيانات والاتساق العلائقي

## استعلامات التقارير

### تم تطوير 7 استعلامات SQL لإنشاء تقارير مهمة:
1. قائمة المرضى مع أطبائهم المعالجين وتواريخ مواعيدهم
2. تقرير الإيرادات حسب القسم
3. تقرير استخدام الأدوية
4. تقرير معدل إشغال الغرف
5. تقرير أداء الأطباء
6. تقرير المرضى الأكثر زيارة للمستشفى
7. تقرير الفواتير غير المدفوعة

## التحديات والدروس المستفادة

### التحديات:
- تصميم العلاقات المعقدة
- ضمان سلامة البيانات
- تطوير استعلامات معقدة

### الدروس المستفادة:
- أهمية التخطيط الجيد
- فوائد التطبيع
- قوة المشغلات والقيود
- مرونة SQL

## الخلاصة

- تم تصميم وتنفيذ نظام قاعدة بيانات متكامل لإدارة المستشفيات
- يغطي النظام جميع جوانب إدارة المستشفى
- تم تطبيق مبادئ تصميم قواعد البيانات الجيدة
- تم تطوير استعلامات SQL لإنشاء تقارير مهمة
- يمكن تطوير النظام مستقبلاً ليشمل المزيد من الوظائف
